# X Search Notes

These are starting points, not rigid templates.

## Collection posture

- Start broad, then narrow.
- Search both narrative terms and concrete nouns.
- Mix topic queries, account queries, and reaction queries.
- Use parallel branches when the topic spans multiple communities.

## Query families worth considering

For a topic like `react compiler`, examples of useful query families are:

- direct topic terms
- launch / release phrasing
- comparison phrasing
- complaint / issue phrasing
- adoption phrasing
- named maintainer or company accounts

Example query ideas:

```text
"react compiler"
"react compiler" lang:en
"react compiler" lang:ja
"react compiler" issue OR bug OR broken
"react compiler" adoption OR rollout
from:vercel "react compiler"
from:reactjs "react compiler"
```

## Practical advice

- Save the broad discovery queries, not only the refined ones.
- Do not discard "boring" counter-evidence if it changes confidence.
- Prefer diverse representative posts over the most-liked posts only.
- Keep tweet URLs or tweet IDs in notes so GPT synthesis can point back to concrete evidence.

## Interpretation boundary

X is useful for:

- speed of reaction
- what people are confused about
- what implementation pain points surface first
- who is shaping the narrative

X is weak for:

- stable specification truth
- usage share claims without stronger evidence
- technical correctness by vote count alone

Use X as live ecosystem evidence, not as the only source of truth.
