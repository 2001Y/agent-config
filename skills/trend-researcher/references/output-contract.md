# Output Contract

This skill keeps the search process flexible, but the saved artifacts are fixed.

## Artifact directory

Write outputs under:

```text
<repo>/_docs/_skills/trend-researcher/<timestamp-pid>/
```

## Required files

### `queries.jsonl`

One line per executed X query.

Recommended fields:

```json
{
  "started_at": "2026-03-10T12:34:56Z",
  "ended_at": "2026-03-10T12:34:58Z",
  "query": "react compiler lang:en",
  "endpoint": "/2/tweets/search/recent",
  "request_params": {
    "max_results": 50,
    "sort_order": "relevancy"
  },
  "result_count": 37,
  "notes": "broad discovery query"
}
```

### `x-search.jsonl`

Raw or lightly normalized X results.

Two acceptable patterns:

- one line per query response
- one line per tweet with query metadata copied down

If you choose one line per tweet, keep the original query text and fetch timestamp on each line.

### `x-search.json`

Script-managed aggregate snapshot of the current `x-search.jsonl`.

- Generated automatically by the wrapper
- Useful to hand to GPT directly
- Do not hand-maintain this file

### `notes.md`

Observational notes only.

Good content:

- repeated phrases
- recurring links
- obvious account clusters
- disagreements
- weak or missing areas in the sample

Avoid final conclusions here.

### `openai.jsonl`

Only when GPT synthesis is used.

Should contain request and response logs in the same style as `coding-confidant`.

### `report.md`

Final user-facing synthesis.

This should be generated by `scripts/generate-report.sh`, not manually assembled by the agent.

## Report requirements

`report.md` must separate:

- what was directly observed on X
- what the agent inferred from those observations

If confidence is low, say so directly.

## Diversity requirement

Before writing the report, check whether the sample is too narrow:

- one launch tweet quoted repeatedly
- one company cluster dominating
- only English or only Japanese when the topic is global
- only positive or only negative sentiment surfaced

If the sample is narrow, run counter-queries and append the new evidence before final synthesis.
