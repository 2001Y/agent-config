# Prompt 派生

これらの断片は `references/default-prompt.md` の差し替え、または task file への追記として使います。

## 設計レビュー寄り

アーキテクチャ境界、依存関係の選択、保守性リスク、移行経路を優先してください。一般論の称賛は避け、トレードオフと未解決の設計判断を明示してください。

## 技術デューデリジェンス寄り

運用リスク、セキュリティ姿勢、リリース成熟度、外部依存、テストカバレッジの兆候、保守負荷を優先してください。確認済みの所見と推測を分けてください。

## 不具合調査寄り

起こりやすい故障点、関連ファイルやサブシステム、現行の公開ドキュメントに記載された外部挙動、次に取るべき最小限のデバッグ手順を優先してください。
